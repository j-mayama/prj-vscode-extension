# 自動レビューループ（Stop hook）

`/codex-review` を毎回手で打つ代わりに、**Claude が応答を終えようとしたタイミングで
未レビューの変更を検知し、自動でレビュー → 選定ダイアログ → 修正まで回す**ための設定。

このファイルは、利用者が自動ループを有効にしたいと言ったときだけ読めばいい。
スキルは hook なしでも `/codex-review` で普通に動く。

## 仕組み

```
Claude が応答を終えようとする
  → Stop hook（scripts/stop-hook.js）が発火
      ├ .codex-review-auto が無い          → 何もしない（通す）
      ├ 差分がない                          → 通す
      ├ 現在の差分がレビュー済み            → 通す
      ├ 現在の差分で既に一度ブロック済み    → 通す（ループ防止）
      └ それ以外                            → block してレビューを指示
  → Claude が codex-review スキルを実行 → ダイアログ → 修正
  → Step 7 で `--mark` を実行し、差分をレビュー済みとして記録
  → Claude が応答を終える → hook は「レビュー済み」なので通す
```

hook 自身は Codex を呼ばない。**「このターンの差分はレビューが要るか」だけを判定する**
軽量な門番で、実際のレビュー・ダイアログ・修正はすべてスキルが担当する。
そのためレビューのロジックは SKILL.md の 1 か所にまとまり、hook は数百 ms で終わる。

## セットアップ

### 1. Stop hook を settings.json に登録する

`~/.claude/settings.json`（全プロジェクトで使う場合）または
`<project>/.claude/settings.json`（そのプロジェクトだけの場合）の `hooks.Stop` に追加する。

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "node \"C:/path/to/skills/codex-review/scripts/stop-hook.js\"",
            "timeout": 30
          }
        ]
      }
    ]
  }
}
```

注意点:

- **パスは必ずスラッシュ（`/`）区切りで書く。** Windows のパスをそのまま
  `C:\Users\...` と貼ると、`\U` は JSON として不正なエスケープなので
  **settings.json 全体が壊れ、そのファイルの設定がすべて無言で無効になる**。
  どうしてもバックスラッシュを使うなら `C:\\Users\\...` と二重にする
- **`"async": true` を付けないこと。** 非同期 hook は撃ちっぱなしになり、
  block の判定結果を返せないのでループが成立しない
- 既に `Stop` hook がある場合は、既存の `hooks` 配列に**追加**する（置き換えない）
- グローバル（`~/.claude/settings.json`）に登録しても、フラグファイルが無いプロジェクトでは
  何も起きない。**登録は一度きり、ON/OFF はフラグファイルで**というのが想定の使い方

### 2. 使いたいリポジトリでフラグファイルを置く

```bash
# ON
touch .codex-review-auto

# OFF
rm .codex-review-auto
```

このファイルがある間だけ自動ループが回る。実装フェーズだけ ON にして、
調査や雑談のときは OFF にするのが実用的。

`.codex-review-auto` は個人の切り替えスイッチなので `.gitignore` に入れておく
（コミットするとチーム全員で ON になる）。gitignore しておくと、フラグ自体が
差分に数えられなくなるという副次的な利点もある。

## 同じプロジェクトで複数セッションを動かすとき

**推奨: セッションごとに git worktree を分ける。**

```bash
git worktree add ../myproj-feature-a -b feature-a
git worktree add ../myproj-feature-b -b feature-b
```

worktree を分ければ、このスキルは何もしなくても完全に分離される。

- 状態ファイルは `git rev-parse --show-toplevel` で分かれるので、
  一方の「レビュー済み」がもう一方に漏れない
- レビュー対象がその worktree の差分だけになるので、他方の書きかけコードを
  レビューしたり修正したりしない
- フラグファイルも worktree ごとなので、片方だけ自動レビューを ON にできる

### 作業ツリーを共有する場合の限界

**1 つのチェックアウトを複数セッションで共有すると、このスキルの有無に関係なく競合する。**
git の差分は「そのツリー全体」でしかないため、セッション A のレビューには
セッション B の書きかけコードが必ず混ざる。A の利用者が B のコードの指摘を採否判断し、
A が B のコードを直してしまう。**これはスキル側では解決できない。**

そのうえで、共有せざるを得ない場合に備えて次の安全策は入れてある。

- **Codex が読んでいない変更をレビュー済みにしない。** `--mark` は
  「レビュー開始前に控えた fingerprint」と現在のツリーが一致するときだけ記録し、
  一致しなければ拒否して終了コード 1 を返す。
  レビュー中に別セッションが割り込んでも、その変更が未レビューのまま
  「レビュー済み」として埋もれることはない（次のターンで再レビューが促される）
- **状態ファイルは書いてから rename する。** 同時書き込みでも、読み手が
  壊れた JSON を読むことはない
- **同じ差分でレビューを起動するのは 1 セッションだけ。** ブロックする前に
  予約ファイルを排他作成（`flag: 'wx'`）するので、複数セッションが同時に停止しても
  勝つのは 1 つだけ。残りはそのターンを普通に終える。
  4 セッション同時停止で block が 1 件になることを実測で確認している

## ループが止まる仕組み

Stop hook には「すでに hook が発火済み」を示すフラグが無い（`stop_hook_active` は
現行のイベントスキーマに存在しない）ため、ガードはこのスクリプトが自前で持っている。

- **差分のフィンガープリント**を取り、状態ファイルに記録する
  （`git status` ＋ `git diff HEAD` ＋ 未追跡ファイルの `git hash-object` の SHA-256）
- **同じ差分では最大 1 回しかブロックしない。** ブロックする前に「試行済み」として
  記録するので、レビューが結果を返せなかった場合（CLI 未導入・未認証・クラッシュ）でも、
  余計なターンは 1 回で済みセッションが詰まらない
- 例外が起きたら**必ず通す**（fail-open）。壊れた hook がセッションを人質にしない

状態ファイルの置き場所: `~/.claude/codex-review-state/<リポジトリのハッシュ>.json`
（リポジトリ内は汚さない）

## トラブルシューティング

| 症状 | 確認すること |
| --- | --- |
| hook が発火しない | フラグファイルがリポジトリのルートにあるか / `node` に PATH が通っているか / `settings.json` の JSON が壊れていないか（パスのバックスラッシュに注意） |
| 毎ターン発火し続ける | スキルの Step 7（`--mark`）が実行されているか。状態ファイルを見て `reviewed` が現在の差分と一致しているか |
| `--mark` が毎回拒否される | レビュー中に作業ツリーが変わっている。レビューの完了を待ってから編集する。別セッションが同じツリーを触っているなら worktree を分ける |
| ループを今すぐ止めたい | `.codex-review-auto` を削除する |

手で試すには:

```bash
# 現在の差分の fingerprint（クリーンなら空行）
node scripts/stop-hook.js --print

# hook 本体（block するときだけ stdout に JSON を出し、通すときは無出力・終了コード 0）
echo '{"hook_event_name":"Stop","cwd":"<repo>","stop_reason":"end_turn"}' | node scripts/stop-hook.js
```
