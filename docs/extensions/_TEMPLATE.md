# テンプレート集（必要なときだけ読む）

拡張を新規に作るときに使うコピペ用テンプレート。CLAUDE.md 本体を薄く保つため、ここに分離している。

---

## フェーズ1：事前調査レポート（作る前に報告する）

```
## 既存拡張の調査結果：<やりたいこと>

### 見つかった拡張
1. <拡張名>（<作者>）
   - インストール数 / 評価 / 最終更新
   - できること：
   - 不足している点：

### 見解
- 既存で足りる？ → はい / いいえ（理由）
- おすすめの進め方 → 作らない / フォーク / 新規

### 判断のお願い
（ユーザーに聞くこと）
```

---

## フェーズ2：拡張ごとのドキュメント（`docs/extensions/<拡張名>.md`）

作ると決まったら、このテンプレートをコピーして拡張名でファイルを作り、実装前に埋める。

```markdown
# <拡張名>

## 目的 / 解決する課題

## フェーズ1：事前調査の結論
- 既存拡張：（あり / なし）
- 参考にした既存拡張：
- 新規で作る理由：

## 使う VSCode API / Contribution Points
- （例）commands / views / languages / debuggers など

## ベストプラクティス（調査結果）
- 参照した公式ドキュメント URL：
- 推奨パターン：
- 避けるべきこと（アンチパターン）：
- パフォーマンス / アクティベーションの注意：

## セキュリティ上の注意（この拡張固有）
- 扱うシークレット / 認証情報：
- Webview の有無と CSP 方針：
- 外部送信するデータ（あれば目的とオプトイン）：

## 実装メモ
- 主要ファイル：
- 未解決の課題 / TODO：
```
